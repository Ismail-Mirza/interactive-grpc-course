Security — TLS & JWT Auth

Secure your gRPC channel with TLS, generate self-signed certificates, and implement JWT authentication interceptors — exactly how FlowGenX authenticates dplane clients connecting to cplane.

⏱ 70 min 🔐 Security 📁 Creates: auth.py + certs/

🔒 TLS — Encrypting the Channel

By default, grpc.insecure_channel() sends data unencrypted. In production, always use TLS. gRPC supports:

🔒
Server-side TLS
Server has a certificate. Client verifies it. Standard HTTPS-style encryption.
🤝
Mutual TLS (mTLS)
Both client AND server have certificates. Each verifies the other. Used in zero-trust B2B scenarios.

Step 1 — Generate Self-Signed Certificates

bashGenerate certs with openssl
mkdir -p grpc-demo/certs && cd grpc-demo/certs

# 1. Generate CA private key and certificate
openssl genrsa -out ca.key 4096
openssl req -new -x509 -days 3650 -key ca.key -out ca.crt \
  -subj "/C=US/O=FlowGenX/CN=FlowGenX-CA"

# 2. Generate server private key and CSR
openssl genrsa -out server.key 4096
openssl req -new -key server.key -out server.csr \
  -subj "/C=US/O=FlowGenX/CN=cplane.flowgenx.local"

# 3. Sign server cert with CA
openssl x509 -req -days 365 -in server.csr \
  -CA ca.crt -CAkey ca.key -CAcreateserial \
  -out server.crt

# For mTLS — also create client cert
openssl genrsa -out client.key 4096
openssl req -new -key client.key -out client.csr \
  -subj "/C=US/O=FlowGenX/CN=dplane-client"
openssl x509 -req -days 365 -in client.csr \
  -CA ca.crt -CAkey ca.key -CAcreateserial \
  -out client.crt

Step 2 — TLS Server

def serve_tls(port=50051, certs_dir='../certs'):
    # Load certificates
    with open(f'{certs_dir}/server.key', 'rb') as f:
        private_key = f.read()
    with open(f'{certs_dir}/server.crt', 'rb') as f:
        certificate_chain = f.read()
    with open(f'{certs_dir}/ca.crt', 'rb') as f:
        root_ca = f.read()

    # Server-side TLS (one-way)
    server_credentials = grpc.ssl_server_credentials(
        [(private_key, certificate_chain)]
    )

    # mTLS: also require client certificate
    # server_credentials = grpc.ssl_server_credentials(
    #     [(private_key, certificate_chain)],
    #     root_certificates=root_ca,
    #     require_client_auth=True  # enforce mTLS
    # )

    server = grpc.server(futures.ThreadPoolExecutor(max_workers=10))
    cplane_pb2_grpc.add_ControlPlaneServicer_to_server(ControlPlaneServicer(), server)
    server.add_secure_port(f'[::]:{port}', server_credentials)  # Note: add_secure_port
    server.start()
    logger.info(f"Secure TLS server started on port {port}")
    server.wait_for_termination()

🎫 JWT Authentication

TLS encrypts the channel. JWT authenticates who is connecting. This is the same pattern used in deployed/cplane/flowgenx_cp_grpc_server/ — the cplane validates JWT tokens from dplane clients.

bashInstall JWT library
pip install PyJWT cryptography
pythongrpc-demo/cplane/auth.py
import jwt
import time
from datetime import datetime, timezone, timedelta

# In production: load from env/vault, rotate regularly
SECRET_KEY = "your-256-bit-secret-change-in-production"
ALGORITHM = "HS256"


def create_token(client_id: str, tenant_id: str, ttl_hours: int = 24) -> str:
    """Generate a JWT token for a dplane client."""
    payload = {
        "sub": client_id,
        "tenant_id": tenant_id,
        "iat": datetime.now(timezone.utc),
        "exp": datetime.now(timezone.utc) + timedelta(hours=ttl_hours),
        "iss": "cplane.flowgenx.ai",
    }
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)


def verify_token(token: str) -> dict:
    """Verify a JWT token and return the decoded payload."""
    try:
        payload = jwt.decode(
            token,
            SECRET_KEY,
            algorithms=[ALGORITHM],
            options={"verify_exp": True}
        )
        return payload
    except jwt.ExpiredSignatureError:
        raise ValueError("Token has expired")
    except jwt.InvalidTokenError as e:
        raise ValueError(f"Invalid token: {e}")
pythonJWT Auth Interceptor (server)
from auth import verify_token

class JWTAuthInterceptor(grpc.ServerInterceptor):
    """Validates JWT bearer tokens on all authenticated endpoints."""

    EXEMPT_METHODS = {'/cplane.ControlPlane/RegisterClient'}

    def intercept_service(self, continuation, handler_call_details):
        if handler_call_details.method in self.EXEMPT_METHODS:
            return continuation(handler_call_details)

        meta = dict(handler_call_details.invocation_metadata)
        auth_header = meta.get('authorization', '')

        if not auth_header.startswith('Bearer '):
            def deny(req, ctx):
                ctx.abort(grpc.StatusCode.UNAUTHENTICATED,
                          "Missing Authorization: Bearer  header")
            return grpc.unary_unary_rpc_method_handler(deny)

        token = auth_header[7:]  # strip "Bearer "
        try:
            payload = verify_token(token)
            # Inject the decoded payload into context
            # (In practice, store in context.peer() or a thread-local)
        except ValueError as e:
            def deny(req, ctx):
                ctx.abort(grpc.StatusCode.UNAUTHENTICATED, str(e))
            return grpc.unary_unary_rpc_method_handler(deny)

        return continuation(handler_call_details)

🔄 Token Refresh Pattern

JWTs expire. The dplane must refresh its token before it expires and reconnect with the new token:

pythonToken refresh in the dplane client
import threading

class DPlaneClient:
    def __init__(self, ...):
        self.auth_token = None
        self.token_expires_at = 0
        self._refresh_lock = threading.Lock()

    def get_valid_token(self):
        """Return a valid token, refreshing if needed."""
        if time.time() > (self.token_expires_at - 300):  # refresh 5 min early
            with self._refresh_lock:
                if time.time() > (self.token_expires_at - 300):
                    self._do_token_refresh()
        return self.auth_token

    def _do_token_refresh(self):
        """Call cplane to get a new token (e.g. via separate REST endpoint)."""
        # Example: call OAuth2 token endpoint
        response = requests.post(
            'https://auth.flowgenx.ai/token',
            data={'client_id': self.client_id, 'client_secret': self.secret}
        )
        data = response.json()
        self.auth_token = data['access_token']
        self.token_expires_at = time.time() + data['expires_in']
        logger.info("Token refreshed successfully")
