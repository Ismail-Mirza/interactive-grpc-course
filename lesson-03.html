🎭 oneof — When a Field Can Be Multiple Types

A oneof field means "exactly one of these fields will be set". It's proto3's way of expressing a discriminated union. This is essential when a message can carry different payload types.

🔮
Used in production control plane services
Real-world streaming message types use oneof payload so a single message type can carry either an API request, a callback, a heartbeat, or a raw JSON payload — without needing separate message types for each case.
proto3cplane.proto
message APIPayload {
  string method = 1;
  string path   = 2;
  string body   = 3;
}

message CallbackPayload {
  string request_id = 1;
  string result     = 2;
  bool   success    = 3;
}

message StreamMessage {
  string      message_id = 1;
  MessageType type       = 2;
  string      client_id  = 3;

  // oneof: only ONE of these will be set in any given message
  oneof payload {
    string          json_payload     = 4;  // raw JSON
    APIPayload      api_payload      = 5;  // API call details
    CallbackPayload callback_payload = 6;  // callback result
  }
}

In Python, you check which field is set with message.WhichOneof('payload'):

pythonUsage
msg = cplane_pb2.StreamMessage(
    message_id="abc123",
    type=cplane_pb2.HEARTBEAT,
    client_id="dplane-1"
    # no payload set for heartbeat
)

# Check which oneof variant is set
field = msg.WhichOneof('payload')
# Returns: None, 'json_payload', 'api_payload', or 'callback_payload'

if field == 'api_payload':
    print(msg.api_payload.path)
elif field is None:
    print("heartbeat - no payload")

🗺️ map — Key-Value Pairs

A map field is proto3's dictionary. Perfect for metadata, headers, or configuration key-value pairs.

proto3example
message MessageWithMetadata {
  string message_id = 1;
  string payload    = 2;

  // map field_name = number;
  map<string, string> metadata    = 3;  // {"trace-id": "abc", "env": "prod"}
  map<string, int32>  retry_counts = 4;  // {"service-a": 3, "service-b": 1}
}
pythonUsing map in Python
msg = cplane_pb2.MessageWithMetadata(
    message_id="xyz789",
    payload="data",
)
# Set map fields like a dict
msg.metadata["trace-id"] = "abc-123"
msg.metadata["env"] = "production"
msg.metadata["dplane-version"] = "2.1.0"

# Read
print(msg.metadata["trace-id"])   # "abc-123"
print(dict(msg.metadata))         # {"trace-id": "abc-123", ...}
⚠️
map keys can't be float/double or bytes
Map key types are limited to scalar types except float, double, and bytes. string is the most common key type. Map fields cannot be repeated.

⌚ Well-Known Types

Google ships a set of pre-defined proto messages called Well-Known Types. Import them to handle common data types that proto3 lacks natively.

ImportTypePython equivalentUse case
google/protobuf/timestamp.protogoogle.protobuf.TimestampdatetimeUTC timestamps
google/protobuf/duration.protogoogle.protobuf.DurationtimedeltaTime durations
google/protobuf/any.protogoogle.protobuf.AnyAny message typeDynamic payloads
google/protobuf/struct.protogoogle.protobuf.StructdictJSON-like data
google/protobuf/wrappers.protoStringValue, Int32Value...Nullable scalarsOptional primitives
proto3Using Timestamp
syntax = "proto3";

import "google/protobuf/timestamp.proto";
import "google/protobuf/struct.proto";

message HeartbeatMessage {
  string                    client_id  = 1;
  google.protobuf.Timestamp sent_at    = 2;  // UTC timestamp
  google.protobuf.Struct    metrics    = 3;  // arbitrary JSON-like data
}
pythonWorking with Timestamp in Python
from google.protobuf.timestamp_pb2 import Timestamp
from datetime import datetime, timezone

# Create a Timestamp from now
ts = Timestamp()
ts.FromDatetime(datetime.now(timezone.utc))

# Use it in a message
hb = cplane_pb2.HeartbeatMessage(
    client_id="dplane-1",
    sent_at=ts,
)

# Convert back to Python datetime
dt = hb.sent_at.ToDatetime(tzinfo=timezone.utc)
print(dt)  # 2024-01-15 10:30:00+00:00

🔒 Backward Compatibility — Schema Evolution

gRPC services live for years. Rules for changing your proto without breaking existing clients:

ChangeSafe?Why
Add a new field✓ SafeOld clients ignore unknown fields
Remove a field⚠ Reserve itOld clients may still send it — reserve the number
Rename a field✓ SafeWire format uses field numbers, not names
Change a field number✗ NeverWire format breaks — old/new clients misread data
Change field type (compatible)⚠ Sometimesint32→int64 is ok; string→int32 is not
Add enum value✓ SafeOld clients get 0 (unknown)
proto3Using reserved
message ClientInfo {
  string client_id  = 1;
  string tenant_id  = 2;
  // string old_field = 3;  ← REMOVED

  // Reserve removed field numbers AND names to prevent reuse
  reserved 3;
  reserved "old_field";

  string version = 4;  // new field added later
}

🛠️ Demo Project — Upgrade cplane.proto

Upgrade your cplane.proto from Lesson 2 with production-grade patterns:

proto3grpc-demo/proto/cplane.proto (upgraded)
syntax = "proto3";

package cplane;

import "google/protobuf/timestamp.proto";

// ─── Enums ───────────────────────────────────────────
enum MessageType {
  UNKNOWN          = 0;
  CLIENT_REQUEST   = 1;
  SERVER_REQUEST   = 2;
  RESPONSE         = 3;
  HEARTBEAT        = 4;
  API_REQUEST      = 5;
  API_RESPONSE     = 6;
}

enum ActionType {
  GET    = 0;
  POST   = 1;
  PUT    = 2;
  DELETE = 3;
  PATCH  = 4;
}

enum ClientStatus {
  ACTIVE       = 0;
  DISCONNECTED = 1;
  ERROR        = 2;
}

// ─── Payload Types ───────────────────────────────────
message APIPayload {
  ActionType action  = 1;
  string     path    = 2;
  string     body    = 3;
  map<string, string> headers = 4;
}

message CallbackPayload {
  string request_id      = 1;
  string result          = 2;
  bool   success         = 3;
  int32  status_code     = 4;
}

// ─── Registration ────────────────────────────────────
message RegisterRequest {
  string client_id  = 1;
  string tenant_id  = 2;
  string auth_token = 3;
  string version    = 4;
  map<string, string> capabilities = 5;
}

message RegisterResponse {
  bool   success    = 1;
  string session_id = 2;
  string message    = 3;
}

// ─── Status ──────────────────────────────────────────
message GetStatusRequest {
  string client_id  = 1;
  string session_id = 2;
}

message GetStatusResponse {
  string                    client_id      = 1;
  ClientStatus              status         = 2;
  google.protobuf.Timestamp last_heartbeat = 3;
  int32                     message_count  = 4;
}

// ─── Streaming ───────────────────────────────────────
message ClientMessage {
  string                    message_id = 1;
  MessageType               type       = 2;
  string                    client_id  = 3;
  google.protobuf.Timestamp sent_at    = 4;
  map<string, string>       metadata   = 5;

  oneof payload {
    string          json_payload     = 6;
    CallbackPayload callback_payload = 7;
  }
}

message ServerMessage {
  string                    message_id  = 1;
  MessageType               type        = 2;
  google.protobuf.Timestamp sent_at     = 3;
  map<string, string>       metadata    = 4;

  oneof payload {
    string     json_payload = 5;
    APIPayload api_payload  = 6;
  }
}

// ─── Service ─────────────────────────────────────────
service ControlPlane {
  rpc RegisterClient  (RegisterRequest)          returns (RegisterResponse);
  rpc GetClientStatus (GetStatusRequest)         returns (GetStatusResponse);
  rpc StreamMessages  (stream ClientMessage)     returns (stream ServerMessage);
}

💪 Exercise

Practice advanced proto3 patterns.

Write Python code that:

1. Creates a ClientMessage with type API_REQUEST, a map metadata with "trace-id": "t-001", and an api_payload with action=POST, path="/api/run", body='{"workflow":"wf-1"}'

2. Creates a heartbeat ClientMessage with no payload but with a Timestamp

3. Prints which oneof variant is set for each message

pythonSolution
import sys
sys.path.insert(0, 'proto')

import cplane_pb2
from google.protobuf.timestamp_pb2 import Timestamp
from datetime import datetime, timezone

# 1. API request message with oneof api_payload
api_msg = cplane_pb2.ClientMessage(
    message_id="msg-001",
    type=cplane_pb2.API_REQUEST,
    client_id="dplane-1",
    api_payload=cplane_pb2.APIPayload(
        action=cplane_pb2.POST,
        path="/api/run",
        body='{"workflow":"wf-1"}',
    )
)
api_msg.metadata["trace-id"] = "t-001"

# 2. Heartbeat with Timestamp, no payload
ts = Timestamp()
ts.FromDatetime(datetime.now(timezone.utc))

hb_msg = cplane_pb2.ClientMessage(
    message_id="msg-002",
    type=cplane_pb2.HEARTBEAT,
    client_id="dplane-1",
    sent_at=ts,
)

# 3. Check oneof
print(api_msg.WhichOneof('payload'))  # 'api_payload'
print(hb_msg.WhichOneof('payload'))   # None (heartbeat has no payload)
