⚠️ gRPC Error Model

gRPC has a richer error model than HTTP. Every RPC call completes with a status — either OK or one of 16 error codes. Each error can carry a message string and optional structured details.

gRPC Error Propagation

dplane Client
→ request →
cplane Server
← StatusCode +
message + details ←
grpc.RpcError

Three layers of error information:

🔢
Status Code
One of 16 standardized codes (INVALID_ARGUMENT, NOT_FOUND, UNAUTHENTICATED...). Machine-readable.
💬
Error Message
Human-readable string. e.g. "client_id is required". Not for programmatic parsing.
📋
Error Details
Structured proto messages: BadRequest, ErrorInfo, RetryInfo, DebugInfo. Rich, parseable details.

🖥️ Server-Side: Raising Errors

pythonSimple abort
def RegisterClient(self, request, context):
    # Simple abort — code + message
    if not request.client_id:
        context.abort(
            grpc.StatusCode.INVALID_ARGUMENT,
            "client_id is required and cannot be empty"
        )

    if request.client_id in banned_clients:
        context.abort(
            grpc.StatusCode.PERMISSION_DENIED,
            f"Client {request.client_id} is banned"
        )

    # Catch unexpected errors and return INTERNAL
    try:
        result = do_registration(request)
        return result
    except DatabaseError as e:
        context.abort(grpc.StatusCode.INTERNAL, f"Database error: {str(e)}")

Rich Error Details with google.rpc.Status

Install pip install grpcio-status googleapis-common-protos to use the rich error model:

pythonRich error with field violations
from grpc_status import rpc_status
from google.rpc import status_pb2, code_pb2, error_details_pb2

def RegisterClient(self, request, context):
    # Collect validation errors
    violations = []
    if not request.client_id:
        violations.append(
            error_details_pb2.BadRequest.FieldViolation(
                field="client_id",
                description="client_id cannot be empty"
            )
        )
    if not request.tenant_id:
        violations.append(
            error_details_pb2.BadRequest.FieldViolation(
                field="tenant_id",
                description="tenant_id is required"
            )
        )

    if violations:
        bad_request = error_details_pb2.BadRequest(field_violations=violations)
        rich_status = status_pb2.Status(
            code=code_pb2.INVALID_ARGUMENT,
            message="Request validation failed",
        )
        rich_status.details.add().Pack(bad_request)

        context.abort_with_status(rpc_status.to_status(rich_status))

    return cplane_pb2.RegisterResponse(success=True, session_id="abc")

📟 Client-Side: Catching Errors

pythonHandling gRPC errors
from grpc_status import rpc_status
from google.rpc import error_details_pb2

def register_with_error_handling(stub, request):
    try:
        response = stub.RegisterClient(request)
        return response

    except grpc.RpcError as e:
        code = e.code()
        message = e.details()

        if code == grpc.StatusCode.INVALID_ARGUMENT:
            # Try to extract rich error details
            status = rpc_status.from_call(e)
            if status:
                for detail in status.details:
                    if detail.Is(error_details_pb2.BadRequest.DESCRIPTOR):
                        bad_req = error_details_pb2.BadRequest()
                        detail.Unpack(bad_req)
                        for v in bad_req.field_violations:
                            print(f"  Field '{v.field}': {v.description}")
            raise ValueError(f"Invalid request: {message}")

        elif code == grpc.StatusCode.UNAUTHENTICATED:
            raise PermissionError("Authentication failed — check your auth_token")

        elif code == grpc.StatusCode.UNAVAILABLE:
            print("Server unavailable, will retry...")
            raise  # let caller handle retry

        elif code == grpc.StatusCode.DEADLINE_EXCEEDED:
            raise TimeoutError(f"RegisterClient timed out")

        else:
            raise RuntimeError(f"gRPC error {code}: {message}")

🔄 Error Handling in Streaming

Streaming errors are slightly different — the error arrives when the iterator is exhausted or raises:

pythonStreaming error handling
def start_streaming(self):
    try:
        stream = self.stub.StreamMessages(
            self._message_generator(),
            timeout=None  # keep alive indefinitely
        )
        for server_msg in stream:
            self._handle_server_message(server_msg)

    except grpc.RpcError as e:
        code = e.code()

        if code == grpc.StatusCode.CANCELLED:
            logger.info("Stream cancelled by server (graceful shutdown)")

        elif code == grpc.StatusCode.UNAVAILABLE:
            logger.warning("Server unavailable — will reconnect")
            # trigger reconnection (see Lesson 10)
            self._schedule_reconnect()

        elif code == grpc.StatusCode.UNAUTHENTICATED:
            logger.error("Auth expired — need to re-register")
            # refresh token and re-register
            self._refresh_and_reconnect()

        else:
            logger.error(f"Stream error {code}: {e.details()}")

    finally:
        logger.info("Stream ended")
⚠️
UNAVAILABLE is retryable — others are not
UNAVAILABLE means a transient failure — retry with backoff. UNAUTHENTICATED means you need to fix credentials first. INVALID_ARGUMENT means your code is wrong — retrying won't help. CANCELLED means server shut down gracefully — reconnect after a delay.

💪 Exercise

Add comprehensive error handling to RegisterClient.

Implement robust error handling:

1. Server: validate that version matches the format X.Y.Z (regex check), return INVALID_ARGUMENT with a BadRequest.FieldViolation if not

2. Server: if a client tries to register twice with same client_id, return ALREADY_EXISTS

3. Client: catch ALREADY_EXISTS and log a warning instead of raising an exception (it just means we're already connected)

4. Client: catch UNAVAILABLE and retry up to 3 times with a 2-second sleep between attempts

pythonSolution
import re, time

# Server
def RegisterClient(self, request, context):
    # Validate version format
    if request.version and not re.match(r'^\d+\.\d+\.\d+$', request.version):
        context.abort(grpc.StatusCode.INVALID_ARGUMENT,
                      "version must be in format X.Y.Z (e.g. 2.1.0)")

    # Check for duplicate
    if request.client_id in connected_clients:
        context.abort(grpc.StatusCode.ALREADY_EXISTS,
                      f"Client {request.client_id} is already registered")

    # ... rest of registration


# Client
def register_with_retry(stub, request, max_retries=3):
    for attempt in range(max_retries):
        try:
            return stub.RegisterClient(request)
        except grpc.RpcError as e:
            if e.code() == grpc.StatusCode.ALREADY_EXISTS:
                logger.warning(f"Already registered: {e.details()}")
                return None  # treat as success
            elif e.code() == grpc.StatusCode.UNAVAILABLE:
                if attempt < max_retries - 1:
                    logger.info(f"Unavailable, retry {attempt+1}/{max_retries}")
                    time.sleep(2)
                    continue
            raise
