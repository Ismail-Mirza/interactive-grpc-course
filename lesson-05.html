Streaming Patterns

Implement all four RPC types — unary, server streaming, client streaming, and bidirectional. Build the StreamMessages RPC that keeps a data plane permanently connected to the control plane.

⏱ 75 min 🌊 4 RPC types 📁 Key: StreamMessages

📡 4 RPC Types — The Full Picture

↔️
Unary
rpc Foo(Req) returns (Res)
One req → one res. Done in Lesson 4.
📤
Server Streaming
rpc Foo(Req) returns (stream Res)
One req → many res (server yields).
📥
Client Streaming
rpc Foo(stream Req) returns (Res)
Many req (iterator) → one res.
🔄
Bidirectional
rpc Foo(stream Req) returns (stream Res)
Many req ↔ many res. Full duplex.

📤 Server Streaming

The server yields multiple responses for a single request. The client iterates over them like a Python iterator.

pythonServer — yields responses
def ListConnectors(self, request, context):
    """Server streams connector info back to client."""
    connectors = ["postgres", "mysql", "s3", "salesforce", "snowflake"]
    for conn in connectors:
        if context.is_active():  # check client is still listening
            yield cplane_pb2.ConnectorInfo(
                name=conn,
                status="available"
            )
        else:
            break  # client disconnected
pythonClient — iterates responses
request = cplane_pb2.ListRequest(tenant_id="tenant-abc")
for connector in stub.ListConnectors(request):
    print(f"Connector: {connector.name} ({connector.status})")

📥 Client Streaming

The client sends a sequence of messages; the server reads all of them and returns one final response.

pythonServer — reads request_iterator
def UploadLogs(self, request_iterator, context):
    """Client streams log entries; server returns summary."""
    count = 0
    for log_entry in request_iterator:
        # Process each log entry as it arrives
        logger.info(f"Log: {log_entry.message}")
        count += 1
    return cplane_pb2.UploadResponse(
        success=True,
        message=f"Received {count} log entries"
    )
pythonClient — generator sends messages
def log_generator():
    for i in range(100):
        yield cplane_pb2.LogEntry(
            message=f"Log line {i}",
            level="INFO",
            timestamp=int(time.time())
        )

response = stub.UploadLogs(log_generator())
print(response.message)  # "Received 100 log entries"

🔄 Bidirectional Streaming — The Key Pattern

Both sides send and receive independently. This is how StreamMessages works in a real control plane — the data plane holds a persistent connection and both sides can send at any time.

🔮
The core pattern in production B2B platforms
In a production B2B control plane, StreamMessages bidirectional RPC is the core connection between the cloud server and each customer-side agent. The data plane sends heartbeats and callback results. The control plane sends API commands. One persistent stream handles all of it.
pythongrpc-demo/cplane/server.py — StreamMessages
import queue
import threading
import uuid
from google.protobuf.timestamp_pb2 import Timestamp
from datetime import datetime, timezone

# Per-client outbound queue: server → dplane
client_queues = {}

class ControlPlaneServicer(cplane_pb2_grpc.ControlPlaneServicer):

    def StreamMessages(self, request_iterator, context):
        """
        Bidirectional streaming RPC.
        - Reads messages FROM the dplane (request_iterator)
        - Yields messages TO the dplane (via outbound queue)
        Both happen concurrently in separate threads.
        """
        client_id = None
        outbound_q = queue.Queue()

        def read_from_client():
            """Thread: reads messages from the dplane."""
            nonlocal client_id
            try:
                for msg in request_iterator:
                    client_id = msg.client_id

                    # Register the client's outbound queue
                    if client_id and client_id not in client_queues:
                        client_queues[client_id] = outbound_q
                        logger.info(f"Stream opened: {client_id}")

                    if msg.type == cplane_pb2.HEARTBEAT:
                        logger.debug(f"Heartbeat from {client_id}")
                        # Echo heartbeat back
                        outbound_q.put(cplane_pb2.ServerMessage(
                            message_id=str(uuid.uuid4()),
                            type=cplane_pb2.HEARTBEAT,
                        ))

                    elif msg.WhichOneof('payload') == 'callback_payload':
                        cb = msg.callback_payload
                        logger.info(f"Callback from {client_id}: req={cb.request_id} success={cb.success}")

            except grpc.RpcError:
                pass  # client disconnected
            finally:
                if client_id and client_id in client_queues:
                    del client_queues[client_id]
                    logger.info(f"Stream closed: {client_id}")
                outbound_q.put(None)  # signal sender to stop

        # Start reader thread
        reader = threading.Thread(target=read_from_client, daemon=True)
        reader.start()

        # Main thread: yield messages from the outbound queue
        while context.is_active():
            try:
                msg = outbound_q.get(timeout=1.0)
                if msg is None:
                    break
                yield msg
            except queue.Empty:
                continue

        reader.join(timeout=2)
pythongrpc-demo/dplane/client.py — stream_messages
import queue
import threading
import time
import uuid
from google.protobuf.timestamp_pb2 import Timestamp
from datetime import datetime, timezone


class DPlaneClient:
    def __init__(self, server_address='localhost:50051', client_id='dplane-1'):
        self.server_address = server_address
        self.client_id = client_id
        self.channel = None
        self.stub = None
        self.session_id = None
        self._send_queue = queue.Queue()
        self._running = False

    def connect(self):
        self.channel = grpc.insecure_channel(self.server_address)
        self.stub = cplane_pb2_grpc.ControlPlaneStub(self.channel)

    def _message_generator(self):
        """Generator: yields messages from the send queue."""
        while self._running:
            try:
                msg = self._send_queue.get(timeout=1.0)
                if msg is None:
                    break
                yield msg
            except queue.Empty:
                continue

    def send_heartbeat(self):
        """Put a heartbeat into the send queue."""
        ts = Timestamp()
        ts.FromDatetime(datetime.now(timezone.utc))
        self._send_queue.put(cplane_pb2.ClientMessage(
            message_id=str(uuid.uuid4()),
            type=cplane_pb2.HEARTBEAT,
            client_id=self.client_id,
            sent_at=ts,
        ))

    def send_callback(self, request_id, result, success=True):
        """Send a callback response to the cplane."""
        self._send_queue.put(cplane_pb2.ClientMessage(
            message_id=str(uuid.uuid4()),
            type=cplane_pb2.RESPONSE,
            client_id=self.client_id,
            callback_payload=cplane_pb2.CallbackPayload(
                request_id=request_id,
                result=result,
                success=success,
            )
        ))

    def start_streaming(self):
        """Start the bidirectional stream and process incoming server messages."""
        self._running = True

        def _heartbeat_loop():
            while self._running:
                self.send_heartbeat()
                time.sleep(5)  # heartbeat every 5 seconds

        # Start heartbeat thread
        threading.Thread(target=_heartbeat_loop, daemon=True).start()

        # Start the bidirectional stream
        stream = self.stub.StreamMessages(self._message_generator())

        try:
            for server_msg in stream:
                self._handle_server_message(server_msg)
        except grpc.RpcError as e:
            logger.error(f"Stream error: {e.code()} {e.details()}")
        finally:
            self._running = False

    def _handle_server_message(self, msg):
        """Process a message received from cplane."""
        if msg.type == cplane_pb2.HEARTBEAT:
            logger.debug("Heartbeat ACK from cplane")
        elif msg.WhichOneof('payload') == 'api_payload':
            api = msg.api_payload
            logger.info(f"API command: {api.action} {api.path}")
            # Execute the API call and send callback
            result = f"executed {api.path}"
            self.send_callback(msg.message_id, result, success=True)
        else:
            logger.info(f"Message from cplane: type={msg.type}")

    def close(self):
        self._running = False
        self._send_queue.put(None)
        if self.channel:
            self.channel.close()

🔑 Key Streaming Rules

RuleWhy
Always check context.is_active() in server loopsClient may disconnect at any time. Checking prevents writing to closed streams.
Use queue.Queue for thread-safe message passinggRPC Python server is synchronous by default. Queues decouple reader and writer threads.
Never block yield with a long computationBlocking the yield loop starves other messages. Push to a queue, yield asynchronously.
Generator must return (not raise) to cleanly close streamFalling off the end of the generator closes the stream gracefully.
Handle grpc.RpcError in client stream loopsServer disconnect raises RpcError — catch it and reconnect (Lesson 10).

def push_api_command(client_id, path, action=cplane_pb2.POST):
    """Push an API command to a connected dplane."""
    q = client_queues.get(client_id)
    if not q:
        logger.warning(f"Client {client_id} not connected")
        return False

    msg = cplane_pb2.ServerMessage(
        message_id=str(uuid.uuid4()),
        type=cplane_pb2.API_REQUEST,
        api_payload=cplane_pb2.APIPayload(
            action=action,
            path=path,
            body='{"trigger": "manual"}',
        )
    )
    q.put(msg)
    logger.info(f"Pushed API command to {client_id}: {action} {path}")
    return True
